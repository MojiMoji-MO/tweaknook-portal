How to Create a Safer One-Way Backup with Robocopy

Published on July 12, 2026 • Updated on July 23, 2026 • By Tweaknook Editorial

Robocopy is included with Windows and can copy folders to an external drive or another location. For a beginner backup, a one-way copy is safer than mirror mode because it does not automatically delete destination files that are missing from the source.

Important: Do not use /MIR unless you fully understand it. Mirror mode can delete files from the destination when they no longer exist in the source.

Step 1: Choose the Source and Destination

In this example, the source is the Windows Documents folder and the destination is an external drive:

  • Source: C:\Users\YourName\Documents
  • Destination: D:\ExternalBackup\Documents

Replace both paths with locations that exist on your computer.

Step 2: Use a Non-Destructive Copy Command

Open Notepad and paste:

@echo off
robocopy "C:\Users\YourName\Documents" "D:\ExternalBackup\Documents" /E /COPY:DAT /DCOPY:T /Z /R:2 /W:5 /XJ /LOG+:"C:\Users\YourName\robocopy-backup.log"

if errorlevel 8 (
    echo Backup completed with an error. Check the log file.
) else (
    echo Backup completed. Check the log for details.
)
pause

Save the file as run-backup.bat. In Notepad's Save dialog, choose All Files so it does not become run-backup.bat.txt.

What the Main Options Mean

OptionPurpose
/ECopies subfolders, including empty subfolders, without purging destination files
/COPY:DATCopies file data, attributes, and timestamps
/DCOPY:TPreserves directory timestamps
/ZUses restartable mode for interrupted transfers
/R:2 /W:5Retries twice and waits five seconds between retries
/XJExcludes junction points to reduce recursive path problems
/LOG+Appends results to a log file

Step 3: Test the Backup

  1. Create a small test folder with several sample files.
  2. Change the script to use that folder as the source.
  3. Run the batch file.
  4. Open the destination and confirm the files are readable.
  5. Delete one file from the source and run the script again. With this command, the old destination copy should remain.

Step 4: Schedule It with Task Scheduler

Open Task Scheduler, choose Create Basic Task, select a schedule, and set the action to Start a program. Browse to run-backup.bat.

The destination drive must be connected and use the same drive letter when the task runs.

What This Backup Does Not Provide

  • It does not create historical versions of every changed file.
  • It does not protect an external drive that remains connected during malware or ransomware activity.
  • It does not verify every copied file automatically.
  • It should not be your only copy of irreplaceable data.

For important files, keep more than one backup and disconnect at least one copy when it is not being updated.

Preview the Copy Before Running It

Add /L to the command for a dry run. Robocopy will list the actions it would take without copying files:

robocopy "C:\Users\YourName\Documents" "D:\ExternalBackup\Documents" /E /COPY:DAT /DCOPY:T /Z /R:2 /W:5 /XJ /L

Review the source, destination, and listed files. Remove /L only after the preview is correct. This is especially useful when drive letters or destination folders have changed.
