How to Stop Website Notification Spam in Chrome

Published on July 24, 2026 • By Tweaknook Editorial

Website notifications can be useful when they come from a calendar, email service, delivery tracker, or collaboration tool. They become a problem when shopping sites, unfamiliar news pages, and one-time downloads continue sending alerts long after the original visit.

The most reliable solution is not to disable every notification on the computer. A better approach is to review which websites have permission, remove the sites you no longer trust, and reduce future permission prompts. This keeps useful alerts working while stopping most of the noise.

Identify What Is Appearing on the Screen

Chrome notifications, pop-up windows, and browser extensions are controlled in different places. Changing the wrong setting may leave the original problem unchanged.

What you see Likely source Where to check
A small alert near the edge of the desktop Website notification Chrome notification permissions
A new browser window or tab Pop-up or redirect Pop-ups and redirects settings
Ads or toolbars appearing on many unrelated sites Extension or unwanted software Chrome extensions and device security scan
Alerts that continue after Chrome is closed Chrome background activity or operating-system notification Chrome and operating-system notification settings

If the alert displays a website name or domain, it is probably a website notification. Do not click the notification simply to investigate it. Open Chrome settings directly and review the permission there.

Review Allowed Notification Sites on a Computer

Chrome keeps a list of sites that are allowed to send notifications. Reviewing that list is usually faster than waiting for another unwanted alert to appear.

  1. Open Chrome.
  2. Select the three-dot menu in the upper-right corner.
  3. Select Settings.
  4. Open Privacy and security.
  5. Select Site settings.
  6. Select Notifications.
  7. Review the sites listed under the allowed section.

Remove or block sites you do not recognize, no longer use, or do not need to hear from. Keep permissions only for services where a browser notification has a clear purpose.

A useful allowed list might include a work calendar, a web-based email account, or a project-management tool. A site visited once for a coupon, download, or article usually does not need permanent notification access.

Block Notifications from One Website

If you know which site is causing the problem, change only that site's permission.

  1. Open the website in Chrome.
  2. Select the site-information icon beside the address.
  3. Open the site's permissions.
  4. Find Notifications.
  5. Change the permission to Block.
  6. Reload the page if Chrome requests it.

This method is preferable when you still want notifications from other trusted sites. It avoids turning off useful reminders across the entire browser.

Reduce Future Permission Requests

Many notification problems begin when a permission request appears immediately after opening an unfamiliar page. The request may be accepted accidentally while closing another message.

Chrome offers a quieter notification option that makes these requests less intrusive:

  1. Open Settings.
  2. Select Privacy and security.
  3. Open Site settings.
  4. Select Notifications.
  5. Keep notification requests available if needed.
  6. Select the quieter messaging option.

With quieter prompts, a site can still request permission, but the request is less likely to interrupt the page. You can then grant access deliberately when a site provides a genuine reason for sending notifications.

Understand the Difference Between Notifications and Pop-Ups

Blocking notifications does not automatically block pop-up windows. A pop-up opens inside or beside the browser, while a notification is delivered through the browser's notification system.

To review pop-up settings on a computer:

  1. Open Chrome settings.
  2. Select Privacy and security.
  3. Select Site settings.
  4. Open Pop-ups and redirects.
  5. Review any sites that are allowed to open pop-ups.

Some legitimate services use pop-ups for payment verification, document previews, or sign-in windows. Block unfamiliar sites first rather than assuming every pop-up permission is harmful.

Stop Website Notifications on Android

Chrome on Android also maintains website notification permissions. Menu wording may vary slightly by device or Chrome version, but the general path is consistent.

  1. Open Chrome on the Android device.
  2. Tap the three-dot menu.
  3. Open Settings.
  4. Select Site settings.
  5. Select Notifications.
  6. Review or disable notification permissions.

For a single website, open the site, tap the site-information control near the address, and review its permissions. On supported devices, an unwanted notification may also include an unsubscribe option directly in the notification panel.

Blocking a website in Chrome is different from disabling all Chrome notifications in Android settings. Turning off the entire Chrome notification category may also hide useful alerts from trusted sites.

What to Do If Notifications Continue

If the website is blocked but alerts continue, work through the following checks in order.

1. Confirm the Domain

Several sites may use similar names. Read the domain shown in the notification and compare it with the domain blocked in Chrome. Blocking one site will not block another domain operated by the same company or advertising network.

2. Check Other Chrome Profiles

Work and personal Chrome profiles keep separate settings. A site blocked in one profile may still be allowed in another. Open each profile that is regularly used and review its notification list.

3. Review Browser Extensions

Open the Chrome extensions page and remove extensions you do not recognize or no longer need. Be cautious with extensions that request permission to read and change data on every website.

Do not remove a required workplace extension without checking with the administrator. Managed browsers may have extensions or policies installed by an organization.

4. Check Operating-System Notifications

Windows, macOS, and Android can control Chrome notifications at the operating-system level. If the browser settings look correct, review the device's notification settings and confirm which application is producing the alert.

5. Run a Security Check

Repeated redirects, unfamiliar extensions, changed search settings, or advertisements appearing outside normal webpages may indicate unwanted software rather than a simple notification permission. Update the browser and operating system, then use the device's trusted security tools.

Do Not Click Suspicious Notification Buttons

A notification may claim that a device is infected, a subscription has expired, or a prize is waiting. A website notification cannot reliably diagnose the security condition of the computer. Treat urgent warnings from unfamiliar domains as untrusted.

  • Do not call phone numbers shown in an unexpected browser alert.
  • Do not install a cleanup tool from the notification.
  • Do not enter passwords or payment details after following the alert.
  • Do not approve additional permissions to make the warning disappear.

Close the alert, open Chrome settings manually, and block the source. If account information may have been entered, change the relevant password through the service's official website and review recent account activity.

A Simple Permission Rule

Before allowing a notification, ask whether the alert must arrive immediately and whether the website is trusted enough to remain connected to your attention.

Notifications are usually reasonable for:

  • Calendar appointments
  • Direct messages from a work platform
  • Security alerts from an account you actively use
  • Time-sensitive delivery updates

Notifications are usually unnecessary for:

  • General news sites
  • Shopping promotions
  • One-time download pages
  • Coupon or prize pages
  • Sites that do not explain what the alerts will contain

Monthly Chrome Permission Check

A short monthly review prevents old permissions from accumulating.

  1. Review the notification allow list.
  2. Review pop-up exceptions.
  3. Remove unused extensions.
  4. Check Chrome's safety recommendations.
  5. Update Chrome when an update is available.

This maintenance is also a good time to reduce browser resource use. The guide on reducing Chrome memory use without closing every tab explains how inactive tabs and browser organization affect everyday performance.

Official Chrome Instructions

Google maintains current instructions for managing Chrome website notifications and blocking or allowing pop-ups. Check those pages if the labels in your version of Chrome differ from the steps above.

Important: A website notification that claims to detect malware, account theft, or an expired security product should not be treated as a trusted diagnosis. Close it and verify the issue through the official application or service.

The Practical Result

The goal is not to block every browser alert. It is to keep a small, deliberate list of trusted sites and remove permissions that no longer serve a useful purpose.

Once unwanted domains are blocked and quieter prompts are enabled, future notification requests become easier to evaluate. The browser remains useful for genuine reminders without allowing one-time websites to compete for attention indefinitely.
