How to Install Multiple Windows Apps with WinGet

Published on July 15, 2026 • Updated on July 23, 2026 • By Tweaknook Editorial

WinGet is Microsoft's Windows package manager. It can search for, install, upgrade, export, and import supported application packages. A short PowerShell loop can install several apps without relying on the && operator.

Note: Installation time varies by internet speed, installer size, administrator requirements, and whether an app requires user input or a restart.

Step 1: Confirm WinGet Is Available

Open PowerShell or Windows Terminal and run:

winget --version

If the command is not found, update or install App Installer through Microsoft Store.

Step 2: Confirm Each Package ID

Search before installing because package IDs can differ from product names:

winget search Chrome
winget search Slack
winget search Zoom

Step 3: Install Several Apps

This PowerShell script runs one installation at a time and works without PowerShell 7 pipeline-chain syntax:

$apps = @(
    "Google.Chrome",
    "SlackTechnologies.Slack",
    "Zoom.Zoom",
    "Notion.Notion",
    "Microsoft.PowerToys"
)

foreach ($app in $apps) {
    Write-Host "Installing $app..."

    winget install `
        --id $app `
        --exact `
        --source winget `
        --accept-source-agreements `
        --accept-package-agreements

    if ($LASTEXITCODE -ne 0) {
        Write-Warning "$app may not have installed correctly."
    }
}

Remove any app you do not need. Some installers may still request administrator permission or additional input.

Upgrade Installed Packages

First review available upgrades:

winget upgrade

Then upgrade supported packages:

winget upgrade --all --accept-source-agreements --accept-package-agreements

Not every installed program is managed by WinGet, so some applications may not appear.

Export an App List

Save a list of packages that WinGet can match:

winget export -o "$env:USERPROFILE\Desktop\MyApps.json" --accept-source-agreements

WinGet may display warnings for applications it cannot match to an available package.

Import the List on Another PC

winget import -i "$env:USERPROFILE\Desktop\MyApps.json" --accept-source-agreements --accept-package-agreements

Importing is a batch installation method, not a complete machine clone. It does not restore application settings, sign-ins, documents, or every program.

Safer Setup Checklist

  • Review the package ID and source before installing.
  • Run PowerShell as administrator only when a package requires it.
  • Keep important documents backed up separately.
  • Restart the computer if an installer requests it.
  • Check failed packages individually instead of repeatedly running the whole list.

Troubleshooting Failed Installs

If one package fails, search for it again and install it separately. This makes the error easier to read:

winget search "Application Name"
winget show --id Publisher.PackageName --exact
winget install --id Publisher.PackageName --exact

Common causes include an outdated package ID, a source that needs refreshing, an installer that requires administrator approval, or an existing installation that WinGet cannot match. You can refresh the sources with:

winget source update

Do not assume that a completed batch installed every application. Review the output and open each required app before treating the setup as finished.
