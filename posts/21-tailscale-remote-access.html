How to Access Your PC Remotely with Tailscale

Published on July 18, 2026 • By Tweaknook Editorial

Tailscale can connect approved devices through a private network so that Windows Remote Desktop does not need to be exposed directly through a public router port. The setup is straightforward, but Remote Desktop requirements and access controls still need to be configured correctly.

What You Need

  • Tailscale installed on the remote PC and the device used to connect.
  • Both devices signed into the same tailnet or granted access through tailnet policy.
  • Remote Desktop enabled on the PC being controlled.
  • A supported Windows edition on the remote PC. Microsoft's built-in RDP host requires Windows Pro or another supported business edition; Windows Home can be used as a client but not as the built-in RDP host.
  • A Windows account with a password and permission to use Remote Desktop.

Step 1: Install Tailscale on Both Devices

Install the official Tailscale client on the remote PC and the connecting device. Sign in through a supported identity provider and confirm that both devices appear in the Tailscale admin console.

Each device receives a private Tailscale IP address, usually in the 100.x.x.x range, and may also receive a MagicDNS name. Access still depends on device authorization, account security, and tailnet access rules.

Step 2: Enable Windows Remote Desktop

On the remote PC, open Settings → System → Remote Desktop and enable the feature. Note the PC name and confirm which user accounts are allowed to connect.

Keep the Windows firewall enabled. Installing and enabling the required software should create the necessary local rules; do not open TCP port 3389 on the public router for this setup.

Step 3: Test Tailscale Connectivity

From the connecting device, confirm that the remote PC is online in Tailscale. You can test the path with:

tailscale ping REMOTE_DEVICE_NAME

Tailscale attempts direct peer-to-peer connections when possible. If network conditions prevent a direct path, traffic can use a relay. Relayed connections can be slower, so performance varies by firewall, NAT type, bandwidth, and latency.

Step 4: Connect with Remote Desktop

  1. Open Remote Desktop Connection or the current Microsoft remote desktop client.
  2. Enter the remote PC's Tailscale IP address or MagicDNS name.
  3. Sign in with an account that is allowed to use Remote Desktop.
  4. Confirm the device identity and test the session before relying on it for urgent work.

Common Problems

The remote PC runs Windows Home

Windows Home does not provide the built-in RDP host. Use a supported Windows edition or choose another remote-access application that can operate over the Tailscale network.

A PIN works locally but not remotely

Remote Desktop may require the actual Windows account password rather than a Windows Hello PIN.

The device appears online but RDP fails

Check that Remote Desktop is enabled, the account is permitted, the PC is awake, and the local firewall allows the service. Organization policies can also block remote sign-in.

The connection is slow

Check whether Tailscale is using a direct or relayed path. Large file transfers and high-resolution remote sessions depend on the upload speed of the remote network as well as the download speed of the local network.

Important: Tailscale reduces the need to expose RDP publicly, but it does not remove the need for strong account security, device updates, access rules, and a tested recovery plan.

For current instructions, review Tailscale's Windows RDP guide and Microsoft's Remote Desktop requirements.
