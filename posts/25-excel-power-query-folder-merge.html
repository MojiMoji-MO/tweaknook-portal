How to Combine Files from a Folder with Excel Power Query

Published on July 22, 2026 • Updated on July 23, 2026 • By Tweaknook Editorial

Power Query can combine files from a folder when the files use a consistent structure. After the query is created, adding another matching file and refreshing the workbook can update the combined result without manual copy-and-paste.

Before combining: Use copies of your source files and confirm that the expected sheet, table, or CSV columns are consistent.

Step 1: Prepare the Folder

Create a dedicated folder such as Raw_Ingestion_Zone. Place only the files that belong in the combined dataset inside it.

For the most predictable result, each file should use:

  • The same file type where possible
  • The same column headers
  • The same table or worksheet structure
  • Consistent date and number formats

Step 2: Connect Excel to the Folder

  1. Open a blank Excel workbook.
  2. Select Data → Get Data → From File → From Folder.
  3. Choose the source folder.
  4. Select Combine & Transform Data.
  5. Choose the sample sheet, table, or file object that contains the data.

Power Query creates the main query and several helper queries used to transform each file.

Step 3: Review the Generated Query

The exact M code varies by file type and Excel version. A generated pattern may include code similar to this:

let
    Source = Folder.Files("C:\Corporate\Raw_Ingestion_Zone"),
    FilteredHiddenFiles = Table.SelectRows(
        Source,
        each [Attributes]?[Hidden]? <> true
    ),
    InvokeTransformFile = Table.AddColumn(
        FilteredHiddenFiles,
        "Transform File",
        each #"Transform File"([Content])
    ),
    RenamedColumns = Table.RenameColumns(
        InvokeTransformFile,
        {{"Name", "Source.Name"}}
    ),
    KeptColumns = Table.SelectColumns(
        RenamedColumns,
        {"Source.Name", "Transform File"}
    ),
    ExpandedData = Table.ExpandTableColumn(
        KeptColumns,
        "Transform File",
        Table.ColumnNames(#"Transform File"(#"Sample File"))
    )
in
    ExpandedData

The nested braces in {{"Name", "Source.Name"}} are required because Table.RenameColumns expects a list of rename pairs.

Dependency note: This example assumes that Excel already created the Transform File function and Sample File helper query through the Combine Files interface. It is not intended as a standalone script for a blank query.

Step 4: Clean and Validate the Data

  • Set each column's data type.
  • Filter out temporary or unwanted files.
  • Check for missing or renamed columns.
  • Keep Source.Name when you need to trace a row back to its file.
  • Compare row counts and totals against the original files.

Step 5: Load and Refresh

Select Close & Load to place the result in a worksheet, or choose Close & Load To to create a connection or add the result to the Data Model.

When a new matching file is added to the folder, open the workbook and select Data → Refresh All. Refresh time depends on the number, size, location, and complexity of the source files.

Common Problems

ProblemWhat to check
New columns do not appearReview the sample-file transformation and expansion step
Errors after a file is addedCheck whether that file uses different headers or data types
Unexpected subfolder files appearFolder.Files also returns files from subfolders; add a folder-path filter if needed
Workbook becomes largeRemove unused columns or load to the Data Model instead of a worksheet

Refresh Checklist

  • Add one new test file and confirm that its rows appear after refresh.
  • Compare the combined row count with the source files.
  • Check that dates, decimals, and leading zeros were not changed unexpectedly.
  • Keep the source filename column until the workflow is stable.
  • Review query errors before sharing the workbook with other users.

Once the checks are reliable, the folder can become a controlled input location for recurring reports.

Official references: Folder.Files and Table.RenameColumns
