How to Find Duplicate Files with PowerShell and Export a Report

Published on July 21, 2026 • Updated on July 23, 2026 • By Tweaknook Editorial

File names and file sizes are not enough to confirm that two files contain the same data. PowerShell can calculate a SHA-256 hash for likely duplicates and export matching groups to a CSV report.

This script does not delete files. Matching content can still be required in more than one folder, so review every path before removing anything.

How the Scan Works

  1. List files recursively.
  2. Group them by file size.
  3. Hash only size groups containing more than one file.
  4. Group matching SHA-256 hashes.
  5. Export all matching paths for review.

PowerShell Script

Change $RootPath to the folder you want to scan:

$RootPath = "C:\Users\YourName\Documents"
$ReportPath = "$env:USERPROFILE\Desktop\Duplicate_Audit_Report.csv"

$files = Get-ChildItem -Path $RootPath -File -Recurse -ErrorAction SilentlyContinue

$sizeCandidates = $files |
    Group-Object -Property Length |
    Where-Object { $_.Count -gt 1 } |
    ForEach-Object { $_.Group }

$hashedFiles = foreach ($file in $sizeCandidates) {
    try {
        $hash = Get-FileHash -LiteralPath $file.FullName -Algorithm SHA256 -ErrorAction Stop

        [PSCustomObject]@{
            Hash          = $hash.Hash
            Path          = $file.FullName
            SizeBytes     = $file.Length
            LastWriteTime = $file.LastWriteTime
        }
    }
    catch {
        Write-Warning "Could not hash: $($file.FullName)"
    }
}

$report = foreach ($group in ($hashedFiles | Group-Object -Property Hash | Where-Object { $_.Count -gt 1 })) {
    $ordered = $group.Group | Sort-Object -Property Path

    for ($index = 0; $index -lt $ordered.Count; $index++) {
        [PSCustomObject]@{
            GroupHash     = $group.Name
            Path          = $ordered[$index].Path
            SizeMB        = [math]::Round($ordered[$index].SizeBytes / 1MB, 2)
            LastWriteTime = $ordered[$index].LastWriteTime
            ReviewNote    = if ($index -eq 0) { "First path in sorted group" } else { "Matching content - review manually" }
        }
    }
}

if ($report) {
    $report | Export-Csv -Path $ReportPath -NoTypeInformation -Encoding UTF8
    Write-Host "Report created: $ReportPath"
}
else {
    Write-Host "No matching file hashes were found."
}

Run the Script

  1. Save it as find-duplicates.ps1.
  2. Open PowerShell.
  3. Run the script from its saved location.
  4. Open Duplicate_Audit_Report.csv in Excel.

Large drives can take a long time because hashing reads file contents. Files that are locked or inaccessible may be skipped with a warning.

How to Review the Report

  • Confirm the files open correctly.
  • Check whether software, shortcuts, or projects depend on a specific path.
  • Keep backups before deleting anything.
  • Move suspected duplicates to a temporary review folder before permanent deletion.
  • Do not assume the first listed path is the authoritative original.

Why the Script Does Not Auto-Delete

Identical content does not always mean one copy is unnecessary. A file may be intentionally duplicated across project folders, offline archives, or application directories. The report is designed to support a manual decision rather than make that decision automatically.

Practical Scan Tips

  • Start with one user folder instead of scanning an entire system drive.
  • Exclude backup folders when you already expect intentional duplicate copies.
  • Close applications that may lock active documents.
  • Keep the CSV report until the cleanup has been reviewed.
  • Move files to a temporary holding folder before emptying the Recycle Bin.

Hash comparison is useful for identifying identical content, but the final decision still depends on the purpose and location of each file.
