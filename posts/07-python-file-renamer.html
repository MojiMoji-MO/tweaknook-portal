How to Safely Bulk Rename Files with Python

Published on July 4, 2026 • Updated on July 23, 2026 • By Tweaknook Editorial

Python can rename a large group of files in sequence, but a safe script should do more than attach a new name. It should ignore folders, preserve each file extension, preview the changes, and reduce the chance of filename collisions.

Before starting: Copy a few sample files into a test folder. Do not run a new renaming script on your only copy of important files.

Step 1: Install Python

Download Python from the official Python website. On Windows, enable the option to add Python to PATH during installation. Then open Command Prompt and verify the installation:

python --version

Step 2: Save This Script Outside the Target Folder

Create a file named safe_renamer.py on your Desktop. Replace the example folder path and prefix, then paste this code:

from pathlib import Path

folder = Path(r"C:\Users\YourName\Desktop\ProductPhotos")
prefix = "SUMMER_COLLECTION"

files = sorted(
    (item for item in folder.iterdir() if item.is_file()),
    key=lambda item: item.name.lower()
)

if not files:
    raise SystemExit("No files were found in the selected folder.")

plan = []
for number, old_path in enumerate(files, start=1):
    new_name = f"{prefix}_{number:03d}{old_path.suffix}"
    plan.append((old_path, folder / new_name))

print("Planned changes:")
for old_path, new_path in plan:
    print(f"{old_path.name}  ->  {new_path.name}")

answer = input("Type YES to rename these files: ")
if answer != "YES":
    raise SystemExit("No files were changed.")

# Temporary names prevent source and destination names from colliding.
temporary_plan = []
for number, (old_path, new_path) in enumerate(plan, start=1):
    temp_path = folder / f".__tweaknook_temp_{number}{old_path.suffix}"
    if temp_path.exists():
        raise FileExistsError(f"Temporary file already exists: {temp_path}")
    old_path.rename(temp_path)
    temporary_plan.append((temp_path, new_path))

for temp_path, new_path in temporary_plan:
    if new_path.exists():
        raise FileExistsError(f"Target file already exists: {new_path}")
    temp_path.rename(new_path)

print(f"Renamed {len(files)} files.")

Step 3: Run the Script

Open Command Prompt, move to the folder containing the script, and run:

python safe_renamer.py

The script first prints every planned change. Files are renamed only after you type YES.

What the Script Does Safely

  • Files only: Subfolders are ignored.
  • Extensions preserved: A PNG remains PNG and a PDF remains PDF.
  • Predictable order: Files are sorted by their current names.
  • Preview first: You can stop before any change is made.
  • Two-stage rename: Temporary names reduce collisions when old and new names overlap.

Important Limitations

The script does not understand which photo or document should receive a specific business identifier. It only assigns numbers according to filename order. Review the preview carefully and keep a backup until you confirm the result.

If the process stops during the temporary rename stage, some files may keep names beginning with .__tweaknook_temp_. Do not delete them; rename them manually or restore from your test copy.

Customizing the Name

Change the prefix:

prefix = "CLIENT_DOCUMENT"

Change the number width from 001 to 0001:

new_name = f"{prefix}_{number:04d}{old_path.suffix}"

Use a Small Test Before the Real Folder

Create a temporary folder containing copies of several file types, including names that already resemble the planned output. Run the script, inspect the preview, and open the renamed files afterward. This confirms that the folder path, sort order, prefix, and extensions behave as expected before you use the script on a larger collection.

Keep the original folder unchanged until the renamed set has been checked. The script is designed to reduce common naming mistakes, but it cannot decide which file should receive a particular business label.
