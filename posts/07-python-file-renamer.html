I Wasted 3 Hours Renaming 500 Files. Here is the 5-Line Python Script That Does It in 2 Seconds

Published on July 4, 2026 • By Tweaknook Editorial

Last Thursday, our external marketing agency sent over a massive ZIP file containing 500 new product photos. The problem? Every single file was named something completely useless like "IMG_9921_final_v2_edit.jpg". The e-commerce manager came to my desk and told me I needed to rename all 500 of them to match our internal inventory system (e.g., "SUMMER_COLLECTION_001") by 5:00 PM so they could be uploaded to the website.

I started doing it manually. I clicked the first file, pressed F2, typed the new name, and hit Enter. Then I did the second one. Then the third. After 50 files and a cramping wrist, I looked at the clock. I realized this mind-numbing task was going to take me at least three hours of continuous typing. I stopped what I was doing, opened my code editor, and wrote a simple five-line Python script. Two seconds later, all 500 files were perfectly renamed in sequential order. Here is exactly how you can do it too, even if you have never coded in your life.

You Do Not Need to Be a Developer to Use Python

When normal office workers hear the word "Python," they imagine a dark room with a hacker typing furiously on multiple screens. That is a myth. Python is just a tool, exactly like Excel or a calculator. You do not need a computer science degree to use it for basic office automation.

Before writing my script, I considered downloading a bulk file renaming software from the internet. But let's look at why that is a terrible idea for your productivity and computer security.

Method Speed Safety & Security Risk
Manual F2 Renaming Extremely slow (Hours) High risk of human typo errors.
Free Bulk Renamer Software Fast High risk. Often bundled with adware or malware.
5-Line Python Script Instant (2 Seconds) 100% Safe. You control the exact logic.

Step 1: The One-Time Setup

If you do not have Python installed on your work computer, go to the official Python website and download the latest version. During the installation, there is one crucial step: You must check the box that says "Add Python to PATH" before clicking install. This allows your computer to understand Python commands from any folder. That is it. You are ready to automate.

Step 2: The Magic 5-Line Script

Open a basic text editor on your computer (Notepad on Windows or TextEdit on Mac is perfectly fine). Copy and paste the following block of code directly into the blank document:

import os

folder_path = r"C:\Users\YourName\Desktop\ProductPhotos"
prefix = "SUMMER_COLLECTION_"

for index, filename in enumerate(os.listdir(folder_path)):
    old_file = os.path.join(folder_path, filename)
    new_file = os.path.join(folder_path, f"{prefix}{index+1}.jpg")
    os.rename(old_file, new_file)

Save this file to your desktop and name it renamer.py. Make sure it ends in .py and not .txt.

Step 3: Translating the Code into Plain English

I never want you to run code you do not understand. Let me break down exactly what these five lines are doing behind the scenes so you can customize them for your own projects.

  • Line 1 (import os): This simply wakes up Python's internal file manager. It gives the script permission to look at your computer's folders.
  • Line 3 (folder_path): This is where you tell the script where your messy files are living. You just replace my example path with the actual location of your folder. The r before the quote just tells Python to read the path exactly as it is written.
  • Line 4 (prefix): This is the new name you want to give everything. In my case, it was "SUMMER_COLLECTION_".
  • Line 6 (for loop): The os.listdir command grabs every single file in your folder. The enumerate command is the secret sauce—it automatically assigns a number (1, 2, 3...) to each file as it goes down the list.
  • Line 9 (os.rename): The script takes the old ugly filename, attaches your new prefix to the new number, slaps a ".jpg" at the end, and hits save. It does this 500 times in the blink of an eye.

How to Avoid the "Double Extension" Error

There is one common mistake beginners make when using this script. If your original files already have their extensions showing (like image.jpg) and you accidentally tell the script to add another .jpg, you will end up with files named SUMMER_COLLECTION_1.jpg.jpg. This will break the file.

To prevent this, always run the script on a small batch of 5 test files first. Create a dummy folder, copy 5 files into it, and point your folder_path to that dummy folder. If the result looks perfect, point it to the main folder with the 500 files and let it rip.

Buy Back Your Time

Your brain is highly paid to solve complex business problems, analyze trends, and communicate with clients. You are not being paid to act like a robot pressing the F2 key for three hours straight. The next time you face a massive list of files, data rows, or folders that need organizing, do not reach for your mouse. Reach for a Python script. It might take you ten minutes to set up the first time, but it will save you hundreds of hours over the rest of your career.
