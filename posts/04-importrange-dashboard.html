How to Build a Google Sheets Dashboard with IMPORTRANGE

Published on July 1, 2026 • By Tweaknook Editorial

IMPORTRANGE is useful when one Google Sheets file needs to display data stored in another file. It removes repeated copy-and-paste work, but it should not be described as a true real-time database connection. Updates can be delayed, especially when the source file is large or several import formulas are chained together.

What IMPORTRANGE Does

The function imports a selected range from a source spreadsheet into a destination spreadsheet. The basic syntax is:

=IMPORTRANGE("spreadsheet_url", "Sheet1!A1:D100")

The first argument is the URL of the source spreadsheet. The second argument identifies the sheet tab and cell range to import.

Step 1: Prepare the Source Sheet

Before creating the formula, organize the source data so that the first row contains clear column headings. Avoid merged cells and blank heading rows because they make later filtering and chart creation harder.

  • Keep dates in a consistent date format.
  • Use one column for each type of information.
  • Import only the rows and columns needed by the dashboard.

Step 2: Add the Formula

Open the destination spreadsheet, select an empty cell, and enter a formula similar to this example:

=IMPORTRANGE("https://docs.google.com/spreadsheets/d/EXAMPLE_ID/edit", "Sales!A1:F500")

The first time the two files are connected, Google Sheets may display #REF! with an Allow access button. Select the button while signed in with an account that can open the source file.

Step 3: Build a Summary View

It is usually better to keep the imported data on a separate tab named something like Imported_Data. Build charts, pivot tables, or summary formulas on another tab. This keeps the dashboard easier to maintain.

You can also combine IMPORTRANGE with QUERY to return only selected rows:

=QUERY(
  IMPORTRANGE("https://docs.google.com/spreadsheets/d/EXAMPLE_ID/edit", "Sales!A1:F500"),
  "select Col1, Col2, Col6 where Col3 = 'Completed'",
  1
)

When QUERY is used with imported data, columns are referenced as Col1, Col2, and so on.

Common Problems

The formula stays on Loading

Reduce the imported range. Importing A:Z transfers many unused cells. A range such as A1:F5000 is usually easier to maintain.

The destination shows old values

Imported data does not always refresh immediately. Source calculations, network conditions, and chained imports can delay updates. For time-sensitive reporting, confirm the source values before sending the dashboard.

Users can see more data than expected

Any editor of the destination file may be able to use the existing connection to import other ranges from the same source. Do not use this setup to expose confidential columns to people who should not access them.

Performance Checklist

  • Import the smallest practical range.
  • Summarize data in the source sheet when possible.
  • Avoid long chains where one imported sheet feeds another imported sheet.
  • Use Apps Script or another data tool when scheduled refresh and larger datasets are required.

Google's official documentation notes that IMPORTRANGE is an external data function and that performance can be affected by range size and network conditions. Review the current limits in the IMPORTRANGE documentation.
